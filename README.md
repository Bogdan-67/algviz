# Алгоритм Кэннона — визуализатор

Одностраничное клиентское приложение (React + Vite + TypeScript), которое по
шагам показывает параллельное умножение матриц по **алгоритму Кэннона** на
решётке N×N процессоров. Без бэкенда и сети — всё считается в памяти.

## Запуск

```bash
npm install
npm run dev      # http://localhost:5173
npm test         # юнит-тесты Vitest
npm run build    # сборка в dist/
```

## Что показывает

- **Сетку N×N процессоров** P(i,j): каждый держит операнды `a × b` и накапливает
  один элемент `C(i,j)` как растущую сумму произведений (только что добавленное
  слагаемое подсвечено).
- **Матрицы A и B в текущем состоянии** со сдвигами — видно тороидальное
  «уезжание» элементов за край и возврат с другой стороны (анимация Framer Motion
  по стабильной идентичности элемента).
- **Формирующуюся матрицу C** и **самопроверку**: результат Кэннона на последнем
  шаге поэлементно сравнивается с наивным тройным циклом → индикатор «верно ✓».

## Этапы (в порядке шагов)

0. Исходная раскладка — процессоры держат свои `A(i,j)`, `B(i,j)`, накопители пусты.
1. **Выравнивание (skew)** — строка `i` матрицы A сдвигается влево на `i`,
   столбец `j` матрицы B — вверх на `j`.
2. … N+1. На каждом шаге все процессоры одновременно делают `C += a·b`, затем
   A сдвигается влево на 1, B — вверх на 1 (по тору). Всего N умножений.

Финал: матрица C готова и «разложена» по процессорам — слияние не требуется.

## Управление

- **← / →** — шаг назад / вперёд
- **Пробел** — авто-проигрывание / пауза
- Слайдеры: размер `N` (2–16), скорость, диапазон случайных чисел
- «🎲 Случайные данные», ручное редактирование ячеек A и B (на шаге 0)
- Переключение тёмной / светлой темы

`N` до 8 — полный режим с формулами в ячейках; `N` от 9 до 16 — компактный режим
(только числа), о чём приложение предупреждает.

## Модель снимков

`buildCannonSnapshots(A, B)` заранее вычисляет массив из `N+2` неизменяемых
снимков (исходная раскладка, выравнивание и N шагов умножения). Кнопки и
авто-режим лишь переключают индекс активного снимка — поведение детерминировано,
рассинхронизации нет. На шаге `k` процессор P(i,j) перемножает
`A[i][(i+j+k) mod N]` и `B[(i+j+k) mod N][j]`; за `k = 0..N-1` индекс пробегает
все значения, поэтому сумма равна `Σ A[i][m]·B[m][j] = C[i][j]`.
