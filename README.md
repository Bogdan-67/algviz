# AlgViz — визуализатор параллельных алгоритмов

Три страницы (вкладки сверху):

- **Кэннон** — параллельное умножение матриц (блочный режим, ниже).
- **Параллельный JOIN** — соединение таблиц поверх горизонтально фрагментированных
  (шардированных) данных. Ключи шардируются по непересекающимся диапазонам → каждый
  фрагмент соединяется независимо. Методы: СУБД-соединение (SP JOIN), соединение в
  приложении (черпак / sort-merge), наивное поключевое (RBAR). p параллельных дорожек,
  барьер синхронизации, метрики (время по исполнителям, паралл. = max, послед. = сумма,
  ускорение, теор. максимум = p). Самопроверка против `naiveJoin`. Логика — в
  [src/lib/join.ts](src/lib/join.ts), тесты — [src/lib/join.test.ts](src/lib/join.test.ts).
- **Топологии** — интерактивные сети межсоединений: линейка, кольцо, 2D-решётка, 2D-тор,
  звезда, полносвязный, дерево, гиперкуб. SVG-граф с осмысленной раскладкой под каждую
  топологию (двоичные коды у гиперкуба, заворот-дуги у кольца/тора), живые метрики (узлы,
  связи, степень, диаметр по BFS, бисекция, средняя длина пути), подсветки диаметра /
  бисекции / макс. степени, подсветка соседей по наведению, список подходящих задач и
  сравнительная таблица. Логика — [src/lib/topology.ts](src/lib/topology.ts), тесты —
  [src/lib/topology.test.ts](src/lib/topology.test.ts) (метрики сверены с теорией).

# Алгоритм Кэннона — визуализатор (блочный режим)

Одностраничное клиентское приложение (React + Vite + TypeScript), которое по
шагам показывает параллельное умножение матриц по **алгоритму Кэннона** на
решётке **q×q логических процессоров**. Матрица N×N нарезается на блоки b×b
(b = N / q); каждый процессор P(I,J) держит блок A и блок B и накапливает блок
C(I,J). Без бэкенда и сети — всё считается в памяти.

Это **симуляция**: приложение не запускает p = q² реальных потоков ОС, а в
обычном цикле рисует, что процессоры делали бы, работая параллельно. «Процессор»
здесь — логическая единица модели (один блок), а не обязательно отдельное ядро.

## Режимы (один общий алгоритм)

- **q = N** — поэлементный режим, блок 1×1 (частный случай).
- **1 < q < N** — блочный режим: число шагов алгоритма равно **q**, а не N.
- **q = 1** — один процессор с блоком N×N, обычное последовательное умножение.

Для N = 8 доступны 1, 4, 16 или 64 процессора (q ∈ {1,2,4,8}, блок 8×8 … 1×1).
Допустимые q — это делители N (`divisors(N)`); комбинации с `N % q ≠ 0` запрещены.


## Что показывает

- **Сетку q×q процессоров** P(I,J): каждый держит **блок** A и **блок** B (мини-
  матрицы) и накапливает **блок** C(I,J). Клик по «▸» разворачивает процессор и
  показывает внутреннее умножение блоков b×b (`A · B = product`, затем `C +=`).
- **Матрицы A и B** с сеткой блоков и групповой подсветкой: блоки выравниваются и
  тороидально сдвигаются по решётке q×q (анимация Framer Motion по стабильной
  идентичности блока — видно «уезжание» за край и возврат с другой стороны).
- **Формирующуюся матрицу C** (с границами блоков) и **самопроверку**: результат
  Кэннона на последнем шаге поэлементно сравнивается с наивным тройным циклом →
  индикатор «верно ✓».

## Этапы (в порядке шагов)

0. Исходная раскладка — процессоры держат свои блоки A(I,J), B(I,J), накопители пусты.
1. **Выравнивание (skew) блоков** — строка блоков `I` матрицы A сдвигается влево
   на `I`, столбец блоков `J` матрицы B — вверх на `J`.
2. … q+1. На каждом шаге все процессоры делают `C(I,J) += A·B` (умножение блоков
   b×b), затем блоки A сдвигаются влево на 1, B — вверх на 1 (по тору). Всего **q**
   умножений.

Финал: C собрана из блоков-накопителей процессоров — слияние не требуется.

## Управление

- **← / →** — шаг назад / вперёд · **Пробел** — авто-проигрывание / пауза
- Селектор **числа процессоров** p = q² (только делители N) с показом q×q и b×b
- Слайдеры: размер `N` (2–16), скорость, диапазон случайных чисел
- «🎲 Случайные данные», ручное редактирование ячеек A и B (на шаге 0)
- Разворот одного процессора по клику, переключение тёмной / светлой темы

`N` до 8 — обычные ячейки; `N` от 9 до 16 — компактный режим (предупреждение).

## Модель снимков

`buildCannonSnapshots(A, B, q)` (q по умолчанию = N) заранее вычисляет массив из
`q+2` неизменяемых снимков (раскладка, выравнивание и q шагов умножения). Кнопки и
авто-режим лишь переключают индекс активного снимка — поведение детерминировано,
рассинхронизации нет. На шаге `k` процессор P(I,J) перемножает блоки
`A[I][(I+J+k) mod q]` и `B[(I+J+k) mod q][J]`; за `k = 0..q-1` индекс блока
пробегает все значения, поэтому сумма блоков равна `Σ_K A[I][K]·B[K][J] = C[I][J]`.
